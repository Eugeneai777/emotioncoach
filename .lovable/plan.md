## 一、Excel 交付物 `xhs_30day_plan.xlsx`

| H | 标签 | 5-8个 #话题 |
| I | 配图建议（3张） | 每张：场景描述+构图+色调+情绪，用 \| 分隔3张 |

## 二、PDF 交付物 `xhs_30day_strategy.pdf`（可落地版，去虚化）

**原则：每章必须包含【做什么/谁来做/什么时候做/拿什么验证】，禁止泛泛而谈。**

### Ch1 商业目标与漏斗（具体数字版）
- 30天目标：曝光 X 万 UV → 9.9 测评 200单 → 399训练营 30单 → 3980 高端 3单
- 单条爆款定义：500赞/100收藏（保底基线）；2000赞为爆款线
- 反推：日均5条 × 30天 = 150条；按20%出爆款率 = 30条爆款撑起流量盘
- ROI 红线：单条成本（AI生成+人工调优）≤ ¥5；CAC ≤ 9.9 单价的 70%

### Ch2 受众画像 × 痛点场景（嵌入图1图2）
- 中年男性：图2的8个具体场景一一对应文案落点（如"陪客户喝完酒回家"→"电量耗尽叙事"）
- 35+女性：图1的多角色透支场景对应文案（"凌晨陪写作业"→"还剩多少自己"）
- 每个场景标注用在D几、A还是B方案

### Ch3 产品矩阵决策（说人话）
- 为何换中场觉醒力替代财富卡点：列3条数据/逻辑（痛点匹配度30%→90%）
- AB纯净化设计：A 直接转化、B 漏斗培育，避免互相干扰
- 不做青少年家长的取舍依据

### Ch4 30天甘特图（按周可执行）
- W1（D1-7）：6种钩子全开测——每天发布时间表（10:00 / 12:30 / 18:00 / 20:00 / 22:00）
- W2（D8-14）：扩量+首轮淘汰（淘汰标准：48h内<200赞）
- W3（D15-21）：Top3 钩子复刻（每个钩子 ×5 变体）
- W4（D22-30）：固化爆款 + 收割（B方案企微转化加密）

### Ch5 AB实验设计
- 唯一变量：A=直推9.9 / B=免费钩子；其他变量（发布时间/标签/配图风格）控制一致
- 成功指标：A方案看 9.9转化率（基线 2%）；B方案看企微加粉率（基线 5%）+ 7日内 399转化率（基线 10%）
- 决策规则：第7天/第14天/第21天三次决策点，明确 kill / scale 标准

### Ch6 数据复盘机制（周会模板）
- 每周一 9:00 复盘会议程（30分钟）：
  1. 上周 Top5 / Bottom5 文案归因（5min）
  2. AB 漏斗数据 + ROI（10min）
  3. 本周 Top3 钩子复刻方案（10min）
  4. 风险词/审核异常（5min）
- 复盘 SQL/查询模板（直接套用 supabase orders 表 + ref 参数）

### Ch7 风险与应对
- 小红书审核红线词表（具体30+词，分医疗/金融/夸大三类）
- 男性话题特殊处理：禁用"性功能/壮阳"，改用"扛造/电量/顶事"
- 限流应急：3小时内无自然流量 → 删稿重发 / 调整封面
- 企微引流合规：评论区话术需用谐音（"欠♀ → 企微"）

## 三、技术执行（MVP高效率）

1. **AI 批量生成**：复用 `knowledge://skill/ai-gateway/scripts/lovable_ai.py`，model=`google/gemini-3-flash-preview`，1 次 prompt 注入完整 system（图1图2痛点 + 合规词表 + JSON schema），分批 30 条/次循环 5 次 = 150 条。带 1.2s 延迟避 429。
2. **配图 prompt 同批生成**：每条文案附带 3 条英文 image prompt（写实人像/场景/情绪向）
3. **Excel**：openpyxl，wrap_text + 列宽 + 公式驱动 ROI
4. **PDF**：reportlab + 内嵌中文字体（Noto Sans CJK），章节内用具体表格而非散文
5. **QA**：
   - Excel 用 recalculate_formulas.py 验证公式
   - PDF 用 pdftoppm 转图逐页核查文字截断/溢出
   - 抽样 10 条文案核对：字段是否完整、CTA 是否带正确 ref、合规词是否触发
6. 输出 `/mnt/documents/`，用 `<lov-artifact>` 交付
