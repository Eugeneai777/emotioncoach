

# /platform-intro 页面设计优化

## 问题诊断

1. **首屏塞满5个卡片**（核心定义 + 痛点 + 改变 + 使命愿景 + 核心价值），信息密度过高
2. **痛点和改变**用相同的列表卡片样式，视觉单调
3. **板块之间只有1px分隔线**，缺乏呼吸感
4. **section间距 py-4 太紧**，移动端阅读疲劳
5. **核心定义卡片不够突出**，没有hero感

## 改动方案

### 1. 首屏 Hero 化（核心定义区）
- 核心定义区改为全宽 hero 风格，增大间距和字号
- 去掉卡片包裹，直接用大字体 + 渐变背景
- 添加副标题"基于领导力的生活教练系统"
- CTA 按钮上移到 hero 区底部，首屏可见

### 2. 痛点区视觉重设计
- 改为 2列网格布局（而非纵向列表），减少占用高度
- 每项用 emoji + 短文案，去掉外层卡片包裹，直接用背景色块
- 标题加大，用更醒目的样式

### 3. "你会得到的改变"精简
- 从5项减到4项（合并"清晰方向"和"持续成长"）
- 改为横向图标卡片网格（2x2），而非纵向列表
- 去掉 desc 详细描述，只保留 emoji + 关键词

### 4. 使命愿景合并为一行
- 从两个卡片改为单卡片内的两列文字，减少视觉噪音

### 5. 全局间距和呼吸感
- section 间距从 `py-4` 增到 `py-6 sm:py-8`
- 分隔线改为 `my-2` 的空白间距（去掉线条）
- 卡片间距统一加大

### 6. 移动端体验
- 四层系统卡片内 icon 和文字间距优化
- 快捷入口区域 badge 增大触控面积
- 底部 CTA 区增加上方留白
