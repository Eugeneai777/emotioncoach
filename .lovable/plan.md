我会把报告命名从「AI分析报告 / AI教练报告」这种泛称，改成围绕用户痛点、主题和结果感的专业名称，让用户看到后觉得“这份报告正是在说我，很值得领取”。

实施方向：

1. 报告命名不再强调“AI教练报告”
   - 不使用泛泛的「AI教练报告」「AI分析报告」作为主名称。
   - 改为围绕具体痛点和主题命名，例如：
     - 「深夜焦虑模式解析报告」
     - 「财富卡点深度定位报告」
     - 「关系沟通盲点洞察报告」
     - 「职场内耗根源分析报告」
     - 「睡眠焦虑触发因素报告」
   - AI 只作为能力背书出现在说明里，不作为报告主标题。

2. 建立“主题 + 专业价值 + 结果感”的报告命名规则
   - 每条内容生成时增加统一字段：
     - 产品/工具标准名：真实产品或工具名称，方便内部对照
     - 赠品名称：用户领取时看到的赠品名
     - 报告名称：痛点主题型专业报告名
     - 报告价值说明：这份报告能帮用户看见什么
     - 行动建议说明：用户拿到报告后能获得什么具体下一步
   - 报告名称必须具备：具体场景、专业感、结果感，不允许只写“AI报告”。

3. 优化 AI 生成提示词
   - 要求 AI 必须根据痛点生成报告名，而不是套模板。
   - 报告名需要满足：
     - 看得出主题：焦虑、财富、关系、职场、睡眠、情绪等
     - 看得出价值：定位、解析、洞察、评估、行动方案等
     - 看得出用户收益：知道自己为什么卡住、下一步怎么做
   - 禁止使用过度医疗化或承诺型词汇，如“诊断”“治愈”“保证解决”。

4. 更新工作台页面展示
   - 卡片和表格中把字段改成更清楚的私域转化结构：
     - 产品/工具名
     - 限时赠品
     - 专业报告名称
     - 报告能看见什么
     - 下一步行动建议
   - 用户看到的赠品文案会先出现真实产品/工具名，再说明报告价值。

5. 更新导出内容
   - CSV / Markdown 导出同步加入：
     - 产品/工具名
     - 赠品名称
     - 专业报告名称
     - 报告价值
     - 行动建议
   - 方便后续统一做小红书、短视频、企微话术和落地页。

示例生成效果：

```text
产品/工具名：财富卡点测评
限时赠品：限时赠送「财富卡点测评」
专业报告名称：财富卡点深度定位报告
报告价值：看见你为什么赚得到却留不住钱，以及背后的核心信念模式
下一步行动建议：给你3个接下来7天可执行的财富觉察动作
```
