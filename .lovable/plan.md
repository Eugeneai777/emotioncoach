可以，方案调整为：短剧分镜图片全部改用 GPT Image 2.0 生成，不再用默认图片模型。GPT Image 2.0 会放在 Lovable Cloud 后端函数里调用，前端只发分镜和角色信息，不暴露密钥。

实施计划：

1. 新增 GPT Image 2.0 分镜生图后端函数
- 新增后端函数：`drama-scene-image-openai`。
- 使用项目里已经配置好的 OpenAI 密钥，不需要你重新提供。
- 后端接收：
  - 剧本标题
  - 整体画风 style
  - 角色设定 characters
  - 当前分镜 scene
  - 图片比例，默认 9:16
  - 参考图 URL，可选
  - 生成模式：单张 / 批量顺序生成
- 后端调用 GPT Image 2.0，拿到图片后上传到项目公开存储，返回稳定图片 URL。

2. 用“角色视觉圣经”保证一致性
每次生成分镜图时，后端会自动拼接一段统一的视觉设定，而不是只用单个分镜 prompt。

统一提示词结构：

```text
Series visual bible:
- fixed art style
- fixed color palette
- fixed lighting style
- fixed lens/camera language
- fixed character appearance
- fixed outfit and recognizable props

Character bible:
角色A：年龄感、脸型、发型、服装、标志物、气质
角色B：年龄感、脸型、发型、服装、标志物、气质

Current scene:
分镜动作、情绪、场景、构图、镜头、光影

Negative consistency rules:
Do not change the character's face, age, hairstyle, outfit, body type, color palette, art style, or identity.
```

这样可以减少每张图里人物变脸、换衣服、换画风的问题。

3. 加入“角色定妆图”机制
为了进一步提高一致性，我会加一个推荐流程：

```text
生成脚本
  ↓
先生成角色定妆图 / 视觉参考图
  ↓
每个分镜都引用同一批角色参考图
  ↓
批量生成分镜图片
```

页面上会新增：
- “生成角色定妆图”按钮
- 每个角色显示一张参考图
- 后续分镜生图自动带上角色参考图

如果你想快速使用，也可以跳过定妆图，直接按角色文字设定生成分镜图；但一致性会弱一些。

4. 每个分镜支持单独生成和批量生成
在当前 `/admin/drama-script` 页面里新增图片生成状态：
- 单个分镜：“生成图片”按钮
- 全部生成：“批量生成全部分镜图片”按钮
- 每个分镜显示：生成中、成功、失败、重试
- 图片生成完成后显示预览图
- 支持打开大图、复制图片链接、下载图片

5. 批量生成时按顺序生成，增强连续性
批量生成不会并发一次性全发，而是按分镜顺序生成：

```text
分镜1 -> 得到图1
分镜2 -> 使用角色定妆图 + 图1作为参考
分镜3 -> 使用角色定妆图 + 图1/图2作为参考
...
```

这样比每张独立生成更容易保持：
- 同一个角色长相一致
- 服装一致
- 场景色调一致
- 短剧画面质感一致

6. 生成好的图片会保存进脚本数据
- 保存脚本时，将角色定妆图 URL 和每个分镜的生成图 URL 写入当前脚本 JSON。
- 下次从“已保存脚本”载入时，图片会自动恢复展示。
- 导出 JSON 时也会包含图片 URL，方便后续剪辑或投喂视频生成工具。

7. 图片驱动视频生成
当前视频生成主要使用 `scene.imagePrompt`。改造后：
- 如果该分镜已有 GPT Image 2.0 生成图，则视频生成优先使用这张图作为首帧/参考图。
- 如果没有图，则保持原来的文字提示词生成视频。

这样最终流程会变成：

```text
短剧脚本
  ↓
GPT Image 2.0 生成每个分镜图
  ↓
用分镜图生成视频片段
  ↓
生成旁白
  ↓
合并剪辑
```

8. 页面宽度和预览稳定性
新增图片预览区会沿用你之前要求的宽度策略：
- `w-full`
- `max-w-full`
- `min-w-0`
- `overflow-hidden`
- 图片 `object-contain`

确保新加的图片预览不会再造成后台页面横向溢出。

技术说明：
- GPT Image 2.0 调用只放在后端函数里。
- 前端不会直接调用 OpenAI，也不会暴露密钥。
- 图片会上传到现有公开文件存储，前端只展示返回的 URL。
- 如果 GPT Image 2.0 返回 429 或额度不足，会在页面 toast 中显示明确中文错误。
- 模型名会集中放在后端常量里，例如 `gpt-image-2`；如果你的 OpenAI 权限实际使用的模型名不同，只需要改这一处。
