

## 重新设计 `/elder-care` — 参照小劲AI轻入口风格

### 问题
当前 `/elder-care` 页面有 7 个区块（Hero、痛点、功能、产品转化、记忆陪伴、子女关怀、信任感），内容冗长复杂。对比小劲AI首页（XiaojinHome），后者结构极简：品牌头 → 语音大按钮 → 3个功能入口 → 1个挑战CTA → 完。

### 方案
参照 XiaojinHome 的布局模式，将 ElderCarePage 精简为一屏轻入口：

```text
┌─────────────────────────┐
│  🏠 有劲生活馆    分享给家人 │  ← 顶部导航栏
├─────────────────────────┤
│       🌿 有劲陪长辈        │  ← 品牌标题 + 副标题
│    让陪伴更简单更温暖       │
├─────────────────────────┤
│                         │
│      ┌──────────┐       │
│      │  💬 聊天  │       │  ← 中心大圆按钮（类似语音按钮）
│      │  陪我聊聊 │       │     点击直达 /elder-care/chat
│      └──────────┘       │
│   像有人在身边陪着你 🌿    │
├─────────────────────────┤
│ ┌──────┐┌──────┐┌──────┐│
│ │ ☀️   ││ 🔔   ││ 😊   ││  ← 3列功能入口卡片
│ │问候  ││提醒  ││心情  ││     （同 XiaojinHome 的 grid-cols-3）
│ └──────┘└──────┘└──────┘│
├─────────────────────────┤
│ ┌─────────────────────┐ │
│ │ ✅ 每日平安打卡      → │ │  ← 横向CTA条（类似成长100天）
│ └─────────────────────┘ │
├─────────────────────────┤
│   有劲AI · 让陪伴更简单   │  ← Footer
└─────────────────────────┘
```

### 文件变更

| 文件 | 变更 |
|------|------|
| `src/pages/ElderCarePage.tsx` | 全面重写，从 565 行精简到约 180 行 |

### 具体实现

**删除的区块**：痛点区、产品转化区（3张卡片）、记忆陪伴区、子女关怀区、信任感区 — 全部移除。

**保留并重构的内容**：

1. **顶部栏**：左侧"有劲生活馆"返回首页，右侧"分享给家人"（复用现有 handleShare）
2. **品牌区**：🌿 + "有劲陪长辈" 大字 + "让陪伴更简单更温暖" 副标题，风格同 XiaojinHome 的 Sparkles 标题
3. **中心大按钮**：120px 暖橙色圆形按钮（带呼吸脉冲动画），图标 MessageCircle，文字"陪我聊聊"，点击直达聊天。配色改为暖橙色系 `from-orange-400 to-amber-500`
4. **3列功能入口**：`grid-cols-3`，每个卡片含 emoji + 标题 + 一行描述，分别导向问候/提醒/心情
5. **平安打卡横条CTA**：渐变背景条，类似"成长100天"样式，导向 `/alive-check`
6. **底部 Footer**：一行小字"有劲AI · 让陪伴更简单"

**视觉风格**：
- 背景 `bg-gradient-to-br from-orange-50 via-amber-50/40 to-white`（同 XiaojinHome）
- 暖橙色系渐变按钮
- 圆角卡片 `rounded-2xl`，轻阴影
- 字体大小适当加大（长辈友好，按钮文字 15px，描述 12px）
