

# 飞轮运营页面排版优化方案

## 问题分析

当前页面的三个核心指标卡片（触达、转化、收入）存在以下问题：
- 卡片使用了通用的 `Card` 组件，圆角过大（rounded-2xl）、内边距过多，与管理后台的紧凑风格不协调
- Sparkline 迷你图占据了大量垂直空间但数据为零时无实际意义
- 整体视觉层级不清晰，指标卡片与下方 Tab 内容之间缺乏联系

## 优化方案

### 1. 指标栏改为紧凑的 AdminStatCard 风格
将三个独立的大卡片替换为与管理后台统一的 `AdminStatCard` 样式（小圆角、左侧图标、紧凑布局），同时保留增长率指标作为 subtitle。移除 Sparkline 迷你图（数据量少时无意义，且占空间）。

### 2. 头部信息整合
将"X 个活跃活动 · 总投放 ¥X"信息整合到指标栏中作为第四个卡片，而非单独一行文字。

### 3. 优化后的布局结构

```text
┌─────────────────────────────────────────────────┐
│ 4 个紧凑指标卡（grid-cols-2 md:grid-cols-4）     │
│ [活跃活动] [触达 +增长%] [转化 +增长%] [收入 +增长%] │
└─────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────┐
│ Tab: [1 测评&工具] [2 训练营] [3 绽放] [4 合伙人] │
│ 层级小结行 + 创建按钮 + 活动列表                  │
└─────────────────────────────────────────────────┘
```

## 技术实现

### 修改文件：`src/components/partner/FlywheelGrowthSystem.tsx`

1. **移除** `MiniSparkline` 的导入和使用
2. **替换**三个 `Card` 指标卡为四个使用 `AdminStatCard` 组件的紧凑卡片：
   - 活跃活动（campaigns 数量）
   - 触达（reach + 增长率作为 subtitle）
   - 转化（conversions + 增长率作为 subtitle）
   - 收入（revenue + 增长率作为 subtitle）
3. **移除**顶部独立的活动数量文字行
4. **布局**改为 `grid-cols-2 md:grid-cols-4`，在移动端也保持紧凑的两列排布
5. 保留 `GrowthIndicator` 逻辑，将增长百分比作为 `AdminStatCard` 的 `subtitle` 属性传入

### 预期效果
- 指标区域高度减少约 50%
- 与管理后台其他页面（用户管理、社区管理等）的视觉风格完全统一
- 运营人员一目了然看到关键转化数据
