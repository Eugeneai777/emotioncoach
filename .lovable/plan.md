计划将“转化方式”改成可多选，并让生成、保存、加载、续集都沿用多个转化方式。

实施内容：

1. 前端交互改造
- 将当前单个 `conversionStyle` 状态改为 `conversionStyles` 集合/数组。
- “转化方式”按钮从单选高亮改为多选切换。
- UI 文案增加“可多选”提示，例如：`转化方式（可多选）`。
- 至少保留一个转化方式；如果用户全取消，可自动恢复默认“剧情植入”，或生成前提示选择至少一种。

2. 生成请求支持多选
- 生成 AI短剧时传递 `conversionStyles` 数组，而不是只传一个 `conversionStyle`。
- 续集生成时优先沿用已保存脚本的多选转化方式。
- 为兼容旧数据，如果老脚本只有 `conversion_style` 单值，也会自动转成数组继续使用。

3. 保存与加载兼容
- 保存脚本时，在现有字段基础上保留兼容：
  - 旧字段 `conversion_style` 可继续存第一个选项，避免影响已有数据库结构。
  - 新的多选值可写入脚本数据或已有 JSON 字段中，避免必须改数据库。
- 加载历史脚本时：
  - 如果有多选数据，用多选数据恢复。
  - 如果只有旧的单选字段，用旧值恢复为单选数组。

4. 后端提示词改造
- `drama-script-ai` 后端函数支持 `conversionStyles` 数组。
- 将多个转化方式合并成提示词，例如：
  - 剧情植入 + 角色使用：产品要自然出现在剧情中，并由主角在关键时刻使用解决问题。
  - 剧情植入 + 结尾推荐：剧情中自然埋点，结尾再用字幕/旁白做轻推荐。
- 继续兼容旧参数 `conversionStyle`，避免其他调用路径出错。

5. 一致性检查同步更新
- 续集的一致性检查会把多个转化方式都纳入产品植入一致性判断，避免上一集是剧情植入+角色使用，下一集只剩硬广。

技术细节：
- 主要修改：`src/components/admin/DramaScriptGenerator.tsx`、`supabase/functions/drama-script-ai/index.ts`。
- 不需要新增数据库表；优先用现有 `script_data` JSON 保存多选信息，并保留旧字段兼容。
- 旧脚本不会失效，仍能加载、生成续集。
