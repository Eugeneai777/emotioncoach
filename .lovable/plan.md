

# 评估：/laoge 页面嵌入测评入口的商业价值

## 当前问题

/laoge 页面目前只有一个转化触点——顶部悬浮条直接指向 ¥399 训练营（/promo/zhile-havruta）。对于冷流量用户，从"免费工具"直接跳到 ¥399 付费页面，中间缺少过渡环节，转化阶梯太陡。

## 评估结论：**应该加测评入口，且非常合理**

### 理由一：补全漏斗中间环节

您的转化漏斗设计是「测评 → ¥399训练营 → ¥3980训练营」，但当前 /laoge 页面跳过了测评这一环，直接推训练营。加入测评后路径变为：

```text
当前路径（断层）:
  /laoge 工具卡片 → 悬浮条 → ¥399训练营页面
                     ↑ 跳跃太大，冷流量抗拒

优化路径（阶梯式）:
  /laoge 工具卡片 → 测评入口 → 完成测评 → 结果页推荐训练营 → ¥399
                     ↑ 低门槛        ↑ 已投入时间+看到痛点    ↑ 自然承接
```

### 理由二：两个测评与男性人群高度匹配

- **财富卡点测评**（/wealth-block）：¥9.9，20题6分钟，直击"赚不到钱"的潜意识卡点——与"赚钱""事业"工具卡片的用户需求完全重合
- **中场觉醒力测评**（/midlife-awakening）：6维度30题8分钟，精准定位中年男性的整体困境——与"压力""健康"工具卡片的用户需求完全重合

### 理由三：测评结果页已有训练营推荐机制

两个测评完成后的结果页面，已经内置了向训练营引导的逻辑。用户做完测评看到自己的"诊断报告"后，对训练营的接受度远高于冷启动推销。

## 实施建议

### 位置：Hero 区域下方、工具卡片上方

不替换顶部悬浮条（悬浮条保留给已有认知的回访用户），而是在 Hero 和工具卡片之间新增一个测评入口区块。

### 设计思路

```text
┌─────────────────────────────────┐
│  悬浮条：🔥 职场突围方案 [详情→] │ ← 保留，服务回访/高意向用户
├─────────────────────────────────┤
│  Hero：老哥AI / 有事问老哥       │
├─────────────────────────────────┤
│  ┌──────────────┬──────────────┐│
│  │ 💰 财富卡点   │ 🧭 中场觉醒力 ││ ← 新增：2列测评卡片
│  │ 20题·6分钟   │ 30题·8分钟   ││
│  │ [限时¥9.9]   │ [专业版]     ││
│  └──────────────┴──────────────┘│
├─────────────────────────────────┤
│  💰 老哥，今年怎么赚钱           │ ← 原有工具卡片
│  🏔️ 老哥，我事业卡住了           │
│  😤 老哥，我压力有点大           │
│  🏥 老哥，我身体有点不对         │
└─────────────────────────────────┘
```

### 文案方向

- 区块标题：「先测一测，找到你的卡点」或「老哥建议：先做个体检」
- 强调"诊断"而非"购买"，降低心理门槛

### 技术改动

仅修改 `src/pages/LaogeAI.tsx`：在 Hero 和工具卡片之间新增一个测评入口区块，点击分别跳转 `/wealth-block` 和 `/midlife-awakening`。无需新建组件或修改后端。

### 悬浮条是否保留？

**建议保留**。两者服务不同用户：
- 悬浮条 → 已有认知、高意向回访用户，直达训练营
- 测评入口 → 冷流量、需要被教育的新用户，先体验再转化

两个触点形成"快通道 + 慢通道"的双路径设计，覆盖不同决策阶段的用户。
