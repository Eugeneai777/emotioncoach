

# 知乐专属页面：人群选择 + 产品推荐

## 概述

创建一个新的知乐产品页面 `/zhile-products`，用户先选择自己所属人群，系统根据人群推荐对应的知乐产品组合（胶囊 + 训练营 + 工具）。页面分为两步：人群选择 → 产品列表。

## 页面结构

```text
┌─────────────────────────────┐
│   💊 知乐 · 找到适合你的方案   │  Hero
├─────────────────────────────┤
│  👩‍👧 宝妈  💼 职场  💑 夫妻   │  人群选择网格
│  🎓 青少年  🧭 中年  🌿 长辈   │  (复用 AudienceHub 数据)
├─────────────────────────────┤
│  ▼ 选中人群后展开 ▼           │
│  推荐产品卡片列表             │
│  - 知乐胶囊套餐 (跳转购买)    │
│  - 推荐训练营 (跳转 camp)     │
│  - 免费工具入口 (测评/AI对话)  │
│  - 协同抗压套餐 (跳转 synergy)│
└─────────────────────────────┘
```

## 实现方案

### 1. 新建页面 `src/pages/ZhileProductsPage.tsx`

- 顶部 Hero 区域：知乐品牌视觉，深色背景（复用 ZhilePromoHub 风格）
- 人群选择：6 个人群按钮（数据与 AudienceHub 一致），点击后高亮选中
- 产品推荐区：根据选中人群，展示对应产品卡片列表，每张卡片包含：标题、描述、价格/标签、跳转按钮
- 人群-产品映射关系硬编码在页面中（每个人群 3-4 个推荐产品）

### 2. 人群与产品映射

| 人群 | 推荐产品 |
|------|---------|
| 宝妈 | 知乐胶囊、宝妈AI助手(/mama)、情绪测评(/emotion-health) |
| 职场 | 协同抗压套餐(/promo/synergy)、知乐胶囊、职场训练营 |
| 夫妻 | 知乐胶囊、亲密关系页(/marriage)、沟通训练营 |
| 青少年 | 情绪测评(/emotion-health)、知乐胶囊、学业减压工具 |
| 中年 | 知乐胶囊、中年觉醒页(/laoge)、协同套餐 |
| 长辈 | 知乐胶囊、陪伴聊天(/elder-care/chat)、平安打卡(/alive-check) |

### 3. 路由注册

在 `App.tsx` 添加 `/zhile-products` 路由，lazy 加载。

### 4. 入口集成

- 在 `ZhilePromoHub` 添加一张卡片指向此页面
- 可选：在 AudienceHub 或首页添加入口

### 技术细节

- 纯前端页面，无需数据库变更
- 复用现有组件风格（motion 动画、渐变卡片）
- 产品卡片点击跳转到已有页面路由
- 响应式设计，移动端优先
