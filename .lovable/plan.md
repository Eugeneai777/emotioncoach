

## 优化 /partner/benefits-all 权益对比页

### 改动概要

1. **列标题明确标注"有劲合伙人"** - 当前只显示"初级/高级/钻石"，需改为"有劲初级/有劲高级/有劲钻石"
2. **新增"参与分成产品"行** - 在 matrix 表格中加入一行，展示各等级可分成的产品数量和类型
3. **高级和钻石列底部加"立即升级"按钮** - 点击直接弹出支付弹窗完成升级，无需跳转到其他页面

### 文件改动

**唯一修改文件**: `src/pages/PartnerBenefitsUnified.tsx`

#### 1. 列标题改为"有劲XX合伙人"

```
💪 初级  →  💪 有劲初级
🔥 高级  →  🔥 有劲高级
💎 钻石  →  💎 有劲钻石
```

#### 2. 新增"分成产品"行

从 `commissionableProducts` 配置读取，在 matrix 表格中新增一行：

```
| 分成产品 | 绽放+有劲7款 | 有劲7款 | 有劲7款 | 有劲7款 |
```

包括：365会员、3款训练营、3款合伙人套餐，共7款。

#### 3. 高级/钻石列底部加支付按钮

- 移除底部原有的"升级引导卡片"（跳转 `/partner/youjin-intro`）
- 在 matrix 表格下方，为高级和钻石各放一个升级按钮（仅当用户当前等级低于该等级时显示）
- 点击按钮直接打开 `UnifiedPayDialog` 支付弹窗
- 复用 `YoujinPartnerIntro` 中的支付逻辑：`key: youjin_partner_l2/l3`, `name`, `price`
- 支付成功后跳转回 `/partner`

#### 新增依赖引入

- `UnifiedPayDialog` 组件
- `usePaymentCallback` hook（处理小程序支付回调）
- `commissionableProducts` / `totalCommissionableCount` 配置
- `useState` 用于管理支付弹窗状态
