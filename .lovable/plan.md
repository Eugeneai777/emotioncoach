

## 修正历史补录的 200 点流水时间

### 问题
之前为用户 18588235488 (`07f04ecd-7b1d-441c-8734-a61865f27af0`) 补录的「管理员赠送 +200 点」流水，`created_at` 被设成了 2026/3/12（注册赠送之后），但用户说**实际充值是今天 2026/4/23**，时间显示与事实不符。

### 修复方案

**单条 SQL 数据修正**（无需改代码、无需改函数逻辑）：

将该用户 `quota_transactions` 表中 `source = 'admin_recharge'` 且 `amount = 200` 的那条历史补录记录的 `created_at` 更新为 **2026-04-23 当天的实际充值时间**。

具体取值：
- 用户截图于今日（2026-04-23）查看，充值动作已完成
- 取一个合理时间戳：**2026-04-23 早于今日已记录的最近一笔扣费/退款流水之前**，确保排序后该条 +200 出现在「今天」的最上方
- 同步更新 `balance_after`：因为时间挪到 4-23，应保持当时余额逻辑一致；但展示用余额本就是该笔流水写入瞬间的快照，调整时间不需要重算余额（保持 167 即可，因截图实际余额 167 与该值一致）

### 涉及变更
1. **新增 migration**：`UPDATE quota_transactions SET created_at = '2026-04-23 ...' WHERE user_id = '07f04ecd-7b1d-441c-8734-a61865f27af0' AND source = 'admin_recharge' AND amount = 200;`

### 不变的部分
- `user_accounts` 余额（300 总额 / 167 剩余）不动
- `admin_apply_quota_recharge` 函数本次不动（上次已修复，未来充值会自动写入正确的 `now()` 时间戳）
- 其他流水记录、聚合显示、UI 全部不动

### 验收
1. 用户刷新「点数明细」后，「管理员赠送 +200 点（历史补录）」一条出现在 **2026/4/23**
2. 排序回到列表最上方（今天）
3. 总额度 300、剩余 167 不变
4. 之后任何新的管理员充值，时间将自动以充值瞬间的 `now()` 记录，不再需要手动修正
