

## /mini-app 页面优化方案

### 改动总览

共涉及 2 个文件，纯 UI 调整，不影响现有业务逻辑、路由、购买流程、缓存机制。

### 改动 1：移除底部导航「快捷服务」按钮

**文件**：`src/components/awakening/AwakeningBottomNav.tsx`

- 删除 `quickActions` 数组及相关状态 (`isMenuOpen`)
- 删除遮罩层、弹出菜单的 JSX 和动画
- 删除 `handleCenterClick`、`handleQuickAction` 函数
- 删除右侧「快捷服务」按钮
- 底栏布局从「我的 / 开始对话 / 快捷服务」简化为「我的 / 开始对话」两项，居中 Logo 按钮保持不变
- 移除 `Zap` 图标导入

**不影响项**：语音教练入口、额度检查、支付弹窗、登录引导等全部保留

### 改动 2：已购用户快捷面板 + 见证提取到主页面

**文件**：`src/pages/MiniAppEntry.tsx`

**2a — 已购用户快捷面板**（人群入口下方，欢迎语上方）
- 引入 `useAuth` 获取当前用户
- 组件挂载时查询 `orders` 表是否有 `status='paid'` 记录
- 有则显示一行横向滚动快捷入口：「我的订单」→ `/my-page`、「我的测评」→ `/energy-studio?tab=assessments`、「我的训练营」→ `/camps`
- 未登录或无付费记录的用户完全不显示
- 样式：圆角胶囊按钮，水平排列，手机端可横向滚动，电脑端一行显示

**2b — 精选见证提取到主页面**
- 从现有 `testimonials` 数组中精选 4 条（覆盖宝妈、中年男性、职场、情感场景）
- 在欢迎语下方、折叠区上方，以横向滚动卡片形式展示
- 折叠区内的完整见证列表保留不变（深度浏览用户仍可查看全部）

**不影响项**：
- 6 大人群入口内容、路由、样式不变
- 折叠区场景引导不变
- `mp_openid` 缓存逻辑不变
- `preferred_audience` 跳转逻辑不变
- 个性化欢迎语不变

### 影响评估

| 项目 | 影响 |
|------|------|
| 现有路由 | 无变更 |
| 购买/支付流程 | 无变更 |
| 缓存机制 (openid/preferred_audience) | 无变更 |
| 底部导航语音教练 | 无变更 |
| 手机端兼容 | 快捷面板横向滚动适配小屏；底栏两项布局更宽松 |
| 电脑端兼容 | 快捷面板一行平铺；见证卡片自适应宽度 |
