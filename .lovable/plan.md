

## 问题分析

当前带"测一测"的卡片（职场解压、中年觉醒）使用 `pt-4`（16px 顶部内边距）来为角标腾出空间，导致这两张卡片整体下移，与其他 4 张卡片底部不对齐。截图中红框标注的正是这个问题。

## 专业 UI 评估

您的想法（一个放卡片内、一个放卡片外）会导致两张带角标卡片的处理方式不一致，增加维护复杂度且视觉上仍不统一。

**更优方案**：统一用"角标负偏移"——去掉 `pt-4`，让角标用负定位（`-top-2`）轻微溢出卡片上边缘。这样：
- 所有 6 张卡片底部自然对齐（grid 布局原生行为）
- 角标仍然醒目可见，不遮挡卡片内容
- 手机端和电脑端表现一致
- 代码统一，无需区分两张卡片

```text
 ┌──────┐  ┌─[测一测]┐  ┌──────┐
 │ 宝妈 │  │ 职场解压 │  │ 情侣 │
 │      │  │        │  │      │
 └──────┘  └────────┘  └──────┘
  ← 底部对齐，角标轻微溢出顶部 →
```

## 修改内容

**文件**：`src/pages/MiniAppEntry.tsx`（第 434-448 行）

将带 badge 卡片的包裹容器从：
```tsx
<div key={a.id} className="relative pt-4">
  <button className="absolute top-0 right-0 z-20 ...">
```

改为：
```tsx
<div key={a.id} className="relative">
  <button className="absolute -top-2 right-1 z-20 ...">
```

去掉 `pt-4`，角标改为 `-top-2 right-1`，使其向上溢出卡片边缘约 8px。同时在 grid 容器上加 `overflow-visible` 确保溢出部分不被裁剪。

仅改动 2-3 行 CSS class，不涉及任何业务逻辑或跳转功能。
