

# 注入婚因有道完整知识体系到AI教练 Prompt

## 方案

将文章中萃取的婚因有道品牌背景、课程体系、胜任力模型、培养特色等内容，结构化注入两个后端函数的 System Prompt 中。

## 知识要点萃取（来自文章）

| 维度 | 核心内容 |
|------|----------|
| **行业痛点** | 专业咨询师不足需求1/3；理论与实践脱节；技能无法变现 |
| **品牌使命** | "让每一个人享受婚姻之旅"，参与国家标准制定 |
| **胜任力模型** | 七项核心能力：伦理操守、婚姻理论、评估诊断、咨询干预、关系维护、自我发展、资源协作 |
| **课程体系** | 初级(40%胜任力)→中级(65%)→高级(90%+)，六大模块 |
| **培养目标** | "四有"咨询师：有专业、有能力、有收入、有尊严 |
| **核心理论工具** | 幸福公式、旋涡/进化模型、婚姻基因地图(九种基因)、MGCR识别、MGP进阶、MCM五大法宝、MERT四步流程、七个维度、七项能力 |
| **培养特色** | 实战驱动(案例推演/场景对话/导师反馈)、个人品牌+专业装备双核、全生命周期支持 |
| **支持体系** | 专业发展委员会、校友会、公益督导、"咨深"线上平台、幸福12/幸福筑基计划 |

## 修改文件

### 1. `supabase/functions/marriage-ai-tool/index.ts`

在 quarrel、coach、assessment-result 三种模式的 systemPrompt 末尾追加【婚因有道知识库】段落：

- 品牌背景（使命、专业资质、行业地位）
- 核心理论工具名称（幸福公式、婚姻基因地图、MCM五大法宝等），让AI分析时能引用专业框架
- 平台功能引导：分析结束后自然推荐相关测评或服务（如吵架复盘后推荐"沟通模式测评"；测评报告中推荐"预约专业咨询"）
- 保持现有输出格式不变，知识库作为背景参考

### 2. `supabase/functions/marriage-realtime-token/index.ts`

在 `buildMarriageCoachInstructions` 函数的 Prompt 中，在现有【专业能力】之后增加【平台知识库】段落：

- 精简版品牌背景（语音场景需控制 token 长度）
- 核心理论工具（供对话中自然引用）
- 当用户问"你们是什么机构"/"有什么服务"时能准确回答
- 当对话深入时能引导用户使用平台测评工具或预约咨询
- 保持现有简短回复风格（2-3句）不变

### 注入内容示例（精简版，用于语音）

```
【平台知识库】
婚因有道：以"让每一个人享受婚姻之旅"为使命，深耕婚姻家庭服务20余年，参与国家婚姻家庭咨询师标准制定。
- 培养目标：有专业、有能力、有收入、有尊严的"四有"咨询师
- 七项核心胜任力：伦理操守、婚姻理论、评估诊断、咨询干预、关系维护、自我发展、资源协作
- 核心工具：幸福公式、婚姻基因地图(九种基因)、旋涡/进化模型、MGCR状态识别、MCM冲突五大法宝、婚姻七维度、经营七项能力
- 课程：初级(基础入门)→中级(深化技能)→高级(行业影响力)
- 平台功能：5大婚姻测评、AI吵架复盘器、AI沟通教练、语音婚姻教练、关系日记
- 专业服务：1对1婚姻修复咨询、夫妻沟通辅导、家庭关系辅导、婚姻危机干预

当用户需要更深入帮助时，温和引导使用平台测评或预约专业咨询，不强推。
```

文字工具版本会更详细，包含课程模块细节和理论工具的具体解释。
