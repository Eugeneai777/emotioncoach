结论先说人话：你配置了 `https://wechat.eugenewe.net` 只是解决了“域名允许调用微信接口”这一关，不等于分享卡片一定成功。现在后台签名接口能正常返回签名，说明不是完全没配；更像是前端给微信签名的 URL 和微信实际打开的 URL 不一致，导致 `wx.config` 失败，分享就退化成普通链接。

准备这样修：

1. 修正 JSSDK 签名 URL
   - 当前代码会把 `?ref=share` 等参数从地址栏删掉后再签名。
   - 微信，尤其 iOS 微信，常常要求用“用户最初打开的完整 URL”签名。
   - 改成只去掉 `#hash`，保留 `?ref=share`、推广参数等 query，不再主动改地址栏。

2. 提前加载微信 JSSDK
   - 现在 `jweixin-1.6.0.js` 是异步延迟加载，首屏立刻点右上角分享时可能还没准备好。
   - 改成页面一开始就加载微信 JSSDK，并保留等待机制，减少“第一次分享不是卡片”的概率。

3. 加一层微信内调试日志
   - 在微信浏览器里打印清楚：当前 URL、签名 URL、`wx.ready` 是否成功、`wx.error` 的具体错误。
   - 这样如果修完还失败，能直接看到是 `invalid signature`、图片问题，还是公众号 AppID / ticket 问题。

4. 保持分享卡片内容不变
   - 分享标题、描述、封面图、`wechat.eugenewe.net` 分享链接仍走现有 `DynamicOGMeta` 配置。
   - 只修 JSSDK 初始化和签名逻辑，不改用户前端功能。
