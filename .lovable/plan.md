我会把“续集生成”从普通再生成，升级为“系列圣经 + 承接约束 + 结果校验”的流程，确保人物、关系、剧情悬念、产品植入和画面提示词都有连续性。

## 问题判断
当前续集断裂的核心原因不是单个提示词不够长，而是续集生成时缺少稳定的“系列记忆结构”：

1. 续集只传了上一集完整脚本和最后分镜，但没有把“核心角色档案、未解决悬念、关系状态、关键道具、产品线、画风锚点”提炼成强约束。
2. 前端生成续集时把主题写成“某标题 后续：冲突继续升级”，这会诱导 AI 按新主题重新创作，而不是严格承接上一集。
3. 后端虽然要求输出 consistencyCheck，但目前只是让 AI 自评，缺少程序层面的硬校验与低分提示拦截。
4. 若用户从“脚本库”某一集点续集，系统没有清楚展示“将承接哪一集、上一集结尾是什么、续集钩子是什么”，用户很难判断是不是接对了。

## 优化目标

- 续集必须沿用上一集核心人物姓名、关系、年龄感、外貌、服装/风格关键词。
- 第一幕必须接上一集最后一句台词、最后动作或未解决悬念。
- 新冲突只能“升级原矛盾”，不能换主角、换世界观、换故事类型。
- 结果页明确展示“承接点”和“一致性评分”，低于阈值时提示重新生成。
- 有劲AI模式下，产品线和转化逻辑也要连续，不能上一集情绪测评、下一集突然变成无关硬广。

## 实施计划

### 1. 强化后端续集 Prompt：加入“系列圣经”结构
在 `supabase/functions/drama-script-ai/index.ts` 中新增续集专用上下文提炼逻辑，把上一集脚本整理成明确字段后传给 AI：

- `seriesBible.characters`：角色姓名、身份、关系、外貌、服装、性格、动机、角色状态。
- `seriesBible.plotState`：上一集核心矛盾、已揭示真相、未解决悬念、最后动作/台词。
- `seriesBible.visualAnchors`：画风、角色 imagePrompt 关键词、场景气质。
- `seriesBible.productAnchors`：已出现产品、使用场景、转化方式。
- `continuityContract`：续集硬规则，明确“不允许改名、不允许换身份、不允许重开故事、不允许忽略上一集结尾”。

续集用户提示词会从：

```text
《标题》后续：冲突继续升级
```

改为：

```text
请严格承接上一集最后分镜，创作第N集；本集主题是“上一集悬念的下一步升级”，不得重启故事。
```

### 2. 修改工具返回结构：让续集必须返回“承接说明”
续集结果中新增或强化字段：

```ts
continuityBridge: {
  inheritedFromPrevious: string;   // 本集从上一集继承了什么
  openingConnection: string;       // 第1幕如何接上一集最后分镜
  unresolvedHookCarried: string;   // 延续了哪个悬念
  nextEpisodeHook: string;         // 本集留下的新钩子
}
```

这样页面上能直接看到续集到底接住了哪里，而不是只看 AI 自评。

### 3. 程序层一致性校验：不只依赖 AI 自评
在后端解析生成结果后，增加轻量校验：

- 上一集核心角色姓名是否还在续集 `characters` 或 `scenes.dialogue/action/imagePrompt` 中出现。
- 第1个分镜是否引用上一集最后分镜的关键词、人物、动作或台词。
- 续集角色数量是否异常变动。
- 有劲AI模式下 selected products 是否被延续。

如果校验风险高：

- 自动压低 `consistencyCheck.overallScore`。
- 在 `issues` 中追加具体问题，例如“上一集主角张伟未出现在续集角色设定中”。
- 前端用红色提示“建议重生成”。

### 4. 前端续集生成逻辑修正
在 `src/components/admin/DramaScriptGenerator.tsx` 中调整：

- `generateSequel` 不再把 `theme` 改成“标题后续”，避免诱导生成新故事。
- 传入 `sequelEpisodeNumber`、上一集标题、上一集最后分镜摘要、上一集角色摘要。
- 生成中提示改为“正在承接第X集生成第X+1集”。
- 生成成功后展示“承接上一集：xxx”。

### 5. 脚本库 UI 增加续集判断信息
在已保存脚本卡片或结果区增加轻量提示：

- 当前脚本是第几集。
- 如果点击“续集”，将承接哪一集。
- 上一集最后一句/最后动作摘要。
- 续集一致性评分低于 85 时，按钮提示“建议重生成”。

### 6. 兼容已保存老脚本
不会要求重建数据库，也不会破坏老脚本。

- 老脚本没有 `continuityBridge` 时正常显示。
- 生成新续集时后端会从旧 `script_data` 中自动提取角色和最后分镜。
- 保存逻辑沿用现有 `drama_scripts` 表，不新增表。

## 技术修改文件

- `supabase/functions/drama-script-ai/index.ts`
  - 增加续集上下文提炼。
  - 增加 `continuityBridge` schema。
  - 增加后端一致性校验。
  - 强化续集 prompt，禁止重开故事。

- `src/components/admin/DramaScriptGenerator.tsx`
  - 修正续集请求 body。
  - 展示承接点、一致性评分和风险提示。
  - 优化续集按钮与生成状态文案。

## 验收标准

1. 从第1集生成续集时，第2集必须保留第1集核心角色姓名和关系。
2. 第2集第一分镜必须明显接第1集最后分镜，而不是另起炉灶。
3. 如果 AI 生成结果断裂，页面能明确提示一致性风险，而不是让用户误用。
4. 有劲AI产品植入在续集中保持同一产品线和自然转化逻辑。
5. 不影响普通新脚本生成、主题推荐、保存/载入脚本、分镜图片生成等现有功能。
