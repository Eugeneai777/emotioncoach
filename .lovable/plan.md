## 优化目标

针对【男人有劲状态评估】同时解决三个层面：

1. 评分方向问题：用户每题选“正常”时，结果不能显示成“很差”。
2. 图表可读性问题：售前页和结果页雷达图要更大、更清晰，适合中年男性快速看懂。
3. 多端体验问题：手机端、电脑端都要兼容排版，并保证页面滚动和动画流畅。

## 核心判断

当前这套测评本质是“阻力/风险型评分”：

- 第一项通常是正常状态，分数为 0。
- 后续选项表示疲惫、压力、睡眠、信心等阻力增加，分数递增。
- 数据库配置也已说明：高分代表更多 friction/阻力。

但前端结果页使用通用逻辑，把所有测评都按“分数越高越好”展示，导致：

- 选正常项时总分低，却被视觉呈现成低分。
- 雷达图面积小，看起来像各项都不好。
- 维度条颜色和结论容易反向误导。

所以本次不建议重做题库，而是把结果展示从“原始阻力分”转换成用户能理解的“有劲状态指数”。

## 方案一：结果页从“原始得分”改为“有劲状态指数”

仅针对 `/assessment/male_midlife_vitality` 做专项转换：

```text
原始阻力百分比 = 原始分 / 最高分
有劲状态指数 = 100 - 原始阻力百分比
```

用户看到的是：

- `有劲状态指数 96%`
- `当前阻力较低，底盘还稳`
- `适合趁状态还在，建立恢复节奏`

而不是看到：

- `1/60 分`
- 小雷达图
- 像很差的颜色提示

这样既修复体验错误，也更符合商业转化逻辑：正常用户不会被吓跑，同时会被引导做“预防型恢复”。

## 方案二：雷达图改成“有劲状态雷达”

### 结果页

把结果页雷达图从原始阻力分转换为状态值：

```text
维度状态值 = 100 - 当前维度阻力百分比
```

展示逻辑：

- 越靠外：当前状态越稳。
- 越靠内：越需要优先恢复。
- 标题从“能力雷达”改成“有劲状态雷达”。
- 增加说明：`越靠外代表状态越稳，越靠内代表越需要恢复。`

这样当同事每题都选第一项时：

```text
原始阻力低
状态指数高
雷达面积大
结论偏积极
```

### 售前页

售前页的预览雷达也统一为状态向表达：

- 标题：`报告预览 · 有劲状态雷达示例`
- 说明：`示例展示的是状态指数，越靠外代表越稳。`
- 示例数据避免“压力内耗高到底好不好”的歧义，改成更直观的状态标签：
  - 精力续航
  - 睡眠修复
  - 压力调节
  - 关键时刻信心
  - 关系温度
  - 行动恢复力

## 方案三：放大图表并提升中年男性阅读友好度

### 图表尺寸

优化 `DimensionRadarChart` 的可配置能力：

- 增加大图模式，供售前页和结果页使用。
- 提升外圈半径，让雷达图不再显得缩在中间。
- 增大维度标签字号和线条粗细。
- 6 个维度标签自动换行，减少挤压。

### 结果页主图建议

手机端：

- 雷达图高度约 360px 左右。
- 卡片内边距控制在 14-16px，保证图足够大。
- 标签字号不低于 12px。
- 不堆太多说明，避免首屏压力。

电脑端：

- 结果页内容容器从单列窄卡片升级为更适合桌面的大容器。
- 在较宽屏幕下采用双栏：
  - 左侧：主结论 + 有劲状态雷达。
  - 右侧：六项状态条 + AI 洞察/训练营入口。
- 最大宽度控制，避免内容横向拉得太散。

## 方案四：手机端与电脑端兼容排版

### 手机端优先

微信内访问和手机浏览器是优先场景：

- 保持单列流式布局。
- 重要信息顺序：主结论 → 雷达图 → 六项状态 → 建议 → 训练营入口。
- CTA 按钮保持大点击区域。
- 图表卡片避免横向滚动。
- 降低复杂动画数量，保证低端安卓机也顺滑。

### 电脑端增强

电脑端不简单拉宽手机页，而是做响应式增强：

- `max-w-lg` 调整为移动端小容器、桌面端大容器。
- 桌面端可使用 `lg:grid lg:grid-cols-...`。
- 雷达图在桌面端更大，但不超过合理宽度。
- 右侧信息卡片吸附在视觉上更清楚的位置，让用户一眼看到建议和转化入口。

## 方案五：流畅度优化

为了避免图表变大后页面卡顿，会做以下控制：

1. 雷达图组件只做必要渲染，不引入额外重型依赖。
2. 动画保持轻量：保留关键模块淡入，不对图表做复杂连续动画。
3. 避免在渲染中重复计算状态分，使用局部变量或 memo 化处理展示数据。
4. 图片仍使用 lazy loading，避免售前页首屏阻塞。
5. 桌面端双栏只通过 CSS 响应式完成，不增加复杂 JS 布局监听。
6. 维度列表与图表共用同一份转换后的数据，减少重复计算和状态不一致。

## 方案六：维度条和文案同步修正

结果页“维度得分”改为“六项状态”：

- 展示 `状态 92%`，不再强调 `0/12` 这类原始阻力分。
- 颜色按状态指数判断：
  - 80-100：稳
  - 60-79：可调整
  - 40-59：需留意
  - 0-39：优先恢复
- 对每个维度给一条更用户化的提示，例如：
  - 睡眠修复偏低：`先从入睡前 15 分钟关机开始。`
  - 压力调节偏低：`不是你扛不住，是脑子一直没下班。`

## 方案七：AI 洞察空态优化

截图里“AI 个性化洞察：暂无”会显得产品不完整。优化为：

- 加载中：`AI 正在分析你的结果...`
- 暂未生成：`AI 洞察正在生成中，你可以先查看上方状态雷达和恢复建议。`
- 失败时不显示“暂无”，避免用户觉得报告无效。

如需要，也会补充 AI 提示词，让它明确：

```text
原始分是阻力分，低分代表状态更稳，高分代表恢复阻力更高。
```

## 验收标准

### 场景 1：每题选第一项

应看到：

- 有劲状态指数高。
- 雷达图面积大。
- 六项状态多数为“稳”。
- 结论是“底盘还稳/适合建立恢复节奏”。
- 不再出现“低分很差”的感受。

### 场景 2：每题选最后一项

应看到：

- 有劲状态指数低。
- 雷达图面积小。
- 文案提示优先恢复。
- 不恐吓、不诊断，仍然给出可执行下一步。

### 场景 3：手机端

- 图表清晰，无横向滚动。
- 字号适合微信内阅读。
- 按钮点击区域足够大。
- 页面滚动顺畅。

### 场景 4：电脑端

- 不再只是窄窄一列居中。
- 主结论、雷达、维度解释有更合理的信息分区。
- 大屏不会空，也不会过度拉伸。

### 场景 5：流畅度

- 页面进入不卡顿。
- 图表渲染稳定。
- 动画不过度。
- 图片和图表不会造成明显布局跳动。
