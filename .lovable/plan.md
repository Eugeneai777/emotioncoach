

# 已购用户显示「开始训练」按钮

## 现状

当前逻辑：`isPaidCamp && !hasPurchased` 时进入第 215-239 行分支（显示"查看详情"或"购买"）。已购用户走第 240-271 行分支，已经显示"开始训练"。

**问题在于**：`emotion_stress_7` 的已购判断 `hasPurchased` 是否正确生效。如果 `hasPurchased` 为 true，则不会进入 215 行分支，直接走 240 行的 else 分支，按钮已经是"开始训练"（第 255-259 行）。

## 结论

**当前代码已经实现了你的需求**：
- 已购用户：`hasPurchased = true` → 跳过第 215 行条件 → 进入第 240 行 else → 显示"开始训练"
- 未购/游客：`hasPurchased = false` → 进入第 216 行 `emotion_stress_7` 判断 → 显示"查看详情"

**无需代码改动**。建议你在预览中测试：用已购账号和未购账号分别访问 `/camps`，确认按钮显示是否正确。
