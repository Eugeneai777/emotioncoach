

## 桌面看板分组优化方案

### 核心问题

管理员详情页（IndustryPartnerDetail）的 "运营工具" 组包含 9 个 Tab（活动、教练、测评、组合、文案、营销、渠道、提醒、培训），全部挤在一行，无法体现运营逻辑链路。合伙人仪表板同理。

### 运营逻辑重构

按照合伙人实际运营流程重新分组，形成 **"建内容 → 做推广 → 看数据 → 跟客户"** 的闭环：

```text
┌─────────┬──────────────┬──────────────┬──────────────┬──────────────┐
│  设置   │   业务数据    │   内容建设    │   营销获客    │   客户运营    │
│         │              │              │              │              │
│ 基本信息 │ 收益看板     │ AI教练       │ 创建活动     │ 学员管理      │
│         │ 推广链接     │ 测评         │ AI文案       │ 跟进提醒      │
│         │              │ 组合产品     │ 营销活动     │ 培训中心      │
│         │              │              │ 渠道归因     │              │
├─────────┴──────────────┴──────────────┴──────────────┴──────────────┤
│                        组织商城                                     │
│                  团队成员 / 商城商品 / 商城订单                       │
└────────────────────────────────────────────────────────────────────┘
```

### 桌面端交互方案：两级导航

将单行 TabsList 改为 **两级结构**：
- **第一级**：顶部分组按钮（设置 / 业务数据 / 内容建设 / 营销获客 / 客户运营 / 组织商城），每组 2-4 个子项
- **第二级**：选中分组后，下方展示该组的子 Tab

这样每级最多 6 个按钮，视觉清晰，运营流程一目了然。

### 具体改动

#### 1. IndustryPartnerDetail.tsx
- 重新定义 `group` 分组，将原 `operations`（9项）拆为 `content`（教练/测评/组合）、`marketing`（活动/文案/营销/渠道）、`crm`（学员/提醒/培训）
- 将学员管理从 `business` 移到 `crm` 组
- 桌面端渲染改为两级：第一级用独立的按钮组选择分组，第二级用 TabsList 显示组内子项
- 移动端 Select 下拉保持不变，仅更新分组

#### 2. IndustryPartnerDashboard.tsx（合伙人端）
- 同步更新分组逻辑：内容建设（教练/测评）→ 营销获客（文案/营销/渠道）→ 客户运营（提醒/培训）
- 桌面端同样采用两级导航，保持体验一致

### 涉及文件

| `IndustryPartnerDetail.tsx` | 重构分组定义 + 桌面端两级导航 |
| `IndustryPartnerDashboard.tsx` | 同步分组 + 两级导航 |
