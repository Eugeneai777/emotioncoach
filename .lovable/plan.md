我会继续把当前“分镜提示词直接生成视频”的流程，升级为“人物锁定卡 + 风格锁定 + 参考图优先 + 逐镜头生成 + 合并”的工作流，重点解决 8 个镜头生成后人物一、画风和剧情连续性不稳定的问题。

## 当前现状

页面已经具备这些能力：

1. 脚本生成后会得到角色、分镜、图片提示词、台词、旁白。
2. 已有“生成角色定妆图”功能。
3. 已有“生成分镜图片”功能，并且分镜图片会参考上一张图，帮助画面连续。
4. 已有“生成视频”功能，调用即梦 `jimeng_ti2v_v30_pro`。
5. 已有 5 秒 / 10 秒片段选择。
6. 已有逐镜头生成、批量生成、下载、合并视频功能。
7. 当前视频生成主要使用 `scene.imagePrompt`，如果有分镜图，会把分镜图作为 `image_urls` 传给即梦。

当前短板是：视频提示词没有专门拼接“人物一锁定”和“统一风格锁定”，所以 8 个镜头分别生成时，即梦容易出现人物脸、服装、年龄感、风格漂移。

## 改造目标

把视频生成改成以下稳定流程：

```text
脚本生成
  -> 生成/确认人物一锁定卡
  -> 生成人物一参考定妆图
  -> 生成分镜图片
  -> 每个镜头用：人物一参考图 + 分镜图 + 人物锁定提示词 + 风格锁定提示词 + 当前镜头动作
  -> 即梦生成 5秒/10秒视频
  -> 单镜头可重生成
  -> 全部完成后合并下载
```

## 具体改造

### 1. 增加“人物一锁定卡”

在短剧结果页角色区新增一个“人物一锁定卡”，默认取 `result.characters[0]`。

显示内容包括：

- 人物一姓名
- 人物一描述
- 人物一固定形象提示词
- 已生成的定妆图
- 当前是否已用于视频生成

页面会明确提示：

```text
视频生成将优先使用人物一作为主角锁定参考，所有镜头都会带上该人物的外貌、服装、年龄感和气质约束。
```

### 2. 增加“统一风格锁定卡”

根据当前脚本风格自动生成一段固定风格提示词，例如：

```text
竖屏9:16，现实主义都市短剧风格，电影感光影，低饱和度，真实人物表演，浅景深，情绪张力强，不要卡通，不要二次元，不要字幕，不要水印。
```

如果用户选择的是赛博朋克、动漫、中国风、美式漫画等，也会按对应风格生成不同的锁定文案。

### 3. 重写即梦视频提示词拼接逻辑

新增一个内部方法，比如：

```ts
buildJimengVideoPrompt(scene)
```

它不再只传 `scene.imagePrompt`，而是拼成完整提示词：

```text
【人物一锁定】
人物一姓名、年龄感、外貌、服装、气质、角色描述。保持同一人物，不要改变脸型、年龄、发型、服装和身份。

【统一风格锁定】
当前脚本统一画风、光影、色调、镜头语言。不要改变风格，不要出现字幕、水印、UI文字。

【当前镜头】
镜头编号、镜头类型、动作、台词/旁白情绪、原始分镜提示词。

【连续性要求】
这是同一部短剧的第 N 个镜头，必须延续前后镜头的人物状态、服装、场景气质和情绪，不要新增无关主角。
```

这样即梦每条 10 秒视频都会收到一致的角色与风格约束。

### 4. 视频生成优先使用参考图

调整 `generateSceneVideo(scene)` 的 `image_urls` 选择策略。

优先级：

```text
1. 当前分镜图 sceneImages[sceneNumber]
2. 人物一定妆图 characterImages[0]
3. 其他角色定妆图
```

建议传给即梦：

- 如果当前镜头已有分镜图：传分镜图 + 人物一定妆图
- 如果当前镜头没有分镜图：传人物一定妆图

这样可以兼顾：

- 分镜图控制当前构图
- 人物一定妆图控制主角脸和服装

### 5. 批量生成视频前增加一致性检查

点击“全部生成视频”时，如果没有人物一定妆图或分镜图片，页面给出提示：

```text
建议先生成“人物一参考图”和“全部分镜图片”，这样 8 个镜头的人物与风格更一致。仍然可以继续生成视频。
```

为了不阻断使用，提示不做强制拦截。

### 6. UI 文案优化

把当前按钮和说明改得更明确：

- “全部生成视频”改为“按统一人物/风格生成全部视频”
- 单镜头“生成视频”改为“生成本镜头视频”
- 时长说明增加：

```text
8个镜头 × 10秒 ≈ 80秒成片；如果用于小红书/抖音，建议前后镜头用5秒，中间冲突镜头用10秒。
```

### 7. 增加单镜头提示词预览/复制

在每个分镜下新增“复制即梦提示词”按钮。

用户可以看到实际提交给即梦的完整提示词，方便判断人物和风格是否锁定到位。

### 8. 合并逻辑保持不变

现有合并视频能力已经存在，会继续沿用：

- 已完成的视频按分镜顺序收集
- 如果旁白已生成，会一并传入合并函数
- 输出合并后的 MP4 下载

本次不新增数据库表，不修改自动生成的后端客户端文件，也不破坏现有保存脚本功能。

## 生成建议默认值

为了更适合当前 8 镜头短剧，我会把使用建议写到页面上：

```text
标准短剧：8镜头 × 10秒 = 约80秒
短视频节奏：前2镜头5秒，中间4镜头10秒，最后2镜头5秒 = 约50秒
人物一致性最佳实践：先生成角色定妆图，再生成分镜图片，最后生成视频。
```

## 验收标准

1. 点击单个镜头生成视频时，提交给即梦的提示词包含人物一锁定、风格锁定和当前镜头信息。
2. 如果已生成人物一定妆图，视频生成会自动把该图作为参考图传入。
3. 如果已生成当前分镜图，视频生成会同时使用分镜图作为参考，提升构图连续性。
4. 用户可以复制每个镜头的完整即梦提示词。
5. 批量生成 8 个镜头前，会提示先生成角色定妆图和分镜图以提升一致性。
6. 原有生成脚本、保存脚本、生成图片、生成旁白、合并下载功能不受影响。
