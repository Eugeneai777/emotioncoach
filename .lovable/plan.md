## 二维码方案（关键）

**短链选择**：`https://wechat.eugenewe.net/mama`

- 该域名是项目标准外链域名（见 Core 记忆「External Domain」）
- 已经过端口兼容处理（微信 / 微信小程序 / 浏览器 / 抖音等内置 WebView 均能正确跳转 React Router `/mama` 路由）
- 微信小程序内扫码会被微信识别为 H5 网址直接打开，避免被拦截

**端口兼容保障**：

- ✅ 微信扫一扫：直接打开 `/mama`
- ✅ 微信小程序内识别二维码：长按识别 → 在浏览器打开 `/mama`
- ✅ 普通浏览器/相机：识别 URL 直接跳转
- ✅ 抖音/小红书内置浏览器：URL 跳转后由 SPA 路由处理
- ✅ Lovable SPA fallback 已生效，深链 `/mama` 刷新不会 404
- 二维码内容仅包含纯 URL（不带特殊协议头），最大化各扫码端兼容性

**生成方式**：

- 使用 Node `qrcode` 库（或 `qrcode-terminal`）生成 600×600 px PNG，纠错等级 H（30% 容错，便于中央嵌 logo）
- 中心可叠加 80×80 px 有劲 AI 圆形 logo（不影响识别）
- 海报右下角保留约 320×320 px 二维码白底卡片 + "扫码进入「她AI」" 文案

## 实施步骤

1. **生成视觉底图**：用 `imagegen--generate_image`（premium，1920×1080）产出含手机 mockup、女性人物、粉玫瑰背景、装饰元素的海报底图（**不让 AI 写关键中文文案**，只画视觉）
2. **HTML/Canvas 叠加文案**：用 node 脚本 + `sharp` 或 puppeteer 渲染 HTML，把标题、痛点对照、数据、二维码精准合成到底图上（保证中文 0 错字、字号一致）
3. **生成真实二维码**：node 脚本调 `qrcode` 库生成 `https://wechat.eugenewe.net/mama` 二维码 PNG，合成到海报右下角
4. **QA**：将最终 PNG 用 `image_tools--zoom_image` 分区检查（标题区/数据区/二维码区/手机区），并用手机模拟扫码验证 URL
5. **交付**：`<presentation-artifact path="mama-poster.png" mime_type="image/png">`
