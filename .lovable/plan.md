

# 在「我的」页面增加语音通话点数消耗查询

## 商业架构分析

当前 /my-page 页面结构：账号信息 → 订单信息 → 设置 → 产品体验 → 关于我们

语音通话消耗属于**用量明细**，与「订单信息」（购买记录）互补。建议放在**订单信息之后、设置之前**，作为独立的「语音通话记录」区块。

## 呈现形式

采用与订单信息区块一致的设计语言：

1. **顶部汇总条**：显示本月通话总次数、总消耗点数、总时长
2. **通话记录列表**：默认显示最近 3 条，可展开查看更多
   - 每条记录：教练类型标签 + 通话时长 + 消耗点数 + 日期
   - 教练类型映射中文名（如 vibrant_life_sage → 生活教练）

```text
┌──────────────────────────────────┐
│ 🎙 语音通话记录      本月 12 次   │
│ ────────────────────────────────│
│ 总消耗 96 点 · 总时长 48 分钟     │
├──────────────────────────────────┤
│ 生活教练          消耗 8 点       │
│ 通话 4分12秒     2026-04-16      │
├──────────────────────────────────┤
│ 生活教练          消耗 8 点       │
│ 通话 3分05秒     2026-04-15      │
├──────────────────────────────────┤
│        查看全部记录 ▼             │
└──────────────────────────────────┘
```

## 技术方案

| 步骤 | 内容 |
|------|------|
| 1 | 新建组件 `src/components/VoiceUsageSection.tsx`，从 `voice_chat_sessions` 表查询当前用户记录 |
| 2 | 在 `MyPage.tsx` 订单区块之后引入该组件 |

### 数据源

使用 `voice_chat_sessions` 表（已有 RLS），字段：
- `coach_key` → 教练类型
- `duration_seconds` → 通话时长
- `total_cost` → 消耗点数（每分钟 8 点）
- `billed_minutes` → 计费分钟数
- `created_at` → 通话时间

### 查询逻辑

```sql
SELECT * FROM voice_chat_sessions 
WHERE user_id = ? 
ORDER BY created_at DESC 
LIMIT 20
```

本月汇总通过前端 JS 过滤计算（数据量小，无需额外 SQL）。

### 无数据状态

未登录或无通话记录时显示"暂无语音通话记录"。
