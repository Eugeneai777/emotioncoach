## 一、先质疑（避免改错方向）

### 质疑 1：声音题其实**不是**同一段

我查了数据库：
- 有劲测评 Q4 声音 = `/assessment-media/q4_irritate.mp3`（刺耳环境噪声，测**烦躁阈值**）
- 警报测评 Q3 声音 = `/assessment-media/q7_rain.mp3`（雨声，测**身体能不能放松**）

**两段音频不是同一个文件**，问法和选项也完全不同。可能您在预览时没点开播放，被绿色播放条骗了。

→ **结论：声音题不用换，但选项文案我可以再差异化一点，让两次答题感更明显。**

### 质疑 2：旋转图 + 门图，确实重复，但意图不同

| 题 | 有劲测评意图 | 警报测评意图 |
|---|---|---|
| Q1 旋转图 | 测神经紧张（看晕了 = 神经透支） | 测同上，几乎重叠 |
| Q6 门图（半开门 vs 关门） | 测表现焦虑（推不推门） | 测压抑度（关门后泄气） |

旋转图在两个测评里**测的几乎是同一个东西**，无论意图怎么包装，做过一次的用户第二次会觉得"又来"。这两张图**必须换**。

### 质疑 3："结果页简单" = 渲染器不一样，不是文案少

我对比了代码：
- 有劲测评走的是 `male_midlife_vitality` 专属 4 屏漏斗渲染（`MaleVitalityReportCard` / 维度雷达 / MBTI 标签 / `statusBand` 等几十处定制逻辑）
- 警报测评走的是默认 `standard` 渲染器，只有：分数环 + 等级标签 + 描述 + tips 列表

**所以"简单"是真的简单**——光改文案不够，得在渲染层加东西。

→ 但**完全复刻**有劲的 4 屏漏斗不划算（那是为卖训练营做的转化漏斗，警报测评目标是"加微"，路径短）。我推荐**第三条路**：在 standard 基础上加 3 个增强模块，比"简单"丰富，但不做漏斗。

## 二、优化方案

### 2.1 题目替换（共 3 道）

| # | 旧题（撞图） | 新题（差异化） |
|---|---|---|
| Q1 | 旋转图 → 神经紧张 | **图片题：3 张「家门钥匙」特写（亮 / 暗 / 模糊）→ 选最像你掏钥匙那一刻的心情。**测「回家前那一秒的下意识感受」，是本测评核心场景。|
| Q3 声音 | 雨声 10 秒 | 保留，但选项改为身体定位词（肩 / 胃 / 喉咙 / 眉心紧），跟有劲的"烦躁阈值"完全分开 |
| Q6 门图 | 半开的门 → 关上家门那一刻 | **图片题：3 张「沙发」特写（齐整 / 凹陷 / 堆满外套）→ 选你家此刻最像哪张。**意象更生活、更男性、更不撞图。|

需要生成的新素材：
- `q1_keychain.jpg`（3 把钥匙在玄关柜面，光线由亮转暗）
- `q6_sofa.jpg`（一张深夜客厅沙发的真实照片感）

旋转图、半开门图**继续给有劲测评独占**。

### 2.2 结果页升级（3 个增强模块，加在 standard 之上）

不做 4 屏漏斗。在现有页面**插入 3 个块**：

```text
┌─────────────────────┐
│  分数环 + 颜色档    │ ← 现有
│  描述 + tips        │ ← 现有
├─────────────────────┤
│ 🆕 三维信号雷达图   │ ← 新增 A
│ 躯体 / 沉默 / 亲密  │   3 条柱（不是 5 维雷达，太重）
├─────────────────────┤
│ 🆕 同档位男人画像   │ ← 新增 B
│ "你这一档，10 个里  │   3 句话静态文案 + 1 张氛围图
│  有 6 个 3 个月后…" │   （按颜色档预置 4 套）
├─────────────────────┤
│ 🆕 一句话破局       │ ← 新增 C
│ "今天先做 1 件事："│   从 tips 里抽 1 条放大成卡片
│ 大号字 + 加微钩子   │   + 现有 QR sheet
└─────────────────────┘
```

实现路径：在 `DynamicAssessmentResult.tsx` 里加一个 `isMaleUnspokenCheck` 分支，复用现有 `result.dimensionScores` + `result.primaryPattern`，**不新建渲染器**，约 150 行新增。
