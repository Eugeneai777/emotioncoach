

## 优化 Claim 领取成功页 - 动态入口引导

### 现状问题

领取成功后，页面底部只有两个按钮：「开始体验」和「进入首页」，都导向首页 `/`。用户不知道去哪里使用刚领到的权益。

### 方案

根据 `grantedItems` 的内容，动态生成对应的入口按钮，替换现有的固定按钮。

### 入口映射规则

| 领取内容关键词 | 按钮文案 | 跳转路径 |
|---|---|---|
| 尝鲜会员 | 和AI教练聊聊 | `/coach/awakening` |
| 情绪健康测评 | 开始情绪健康测评 | `/emotion-health` |
| SCL-90 | 开始SCL-90测评 | `/scl90` |
| 财富卡点 | 开始财富卡点测评 | `/wealth-block` |
| 死了吗打卡 | 开启每日打卡 | `/alive-check` |
| 觉察日记 | 写觉察日记 | `/awakening` |
| 情绪SOS | 试试情绪SOS | `/emotion-button` |

### UI 设计

- 将权益列表下方的按钮区改为动态按钮列表
- 第一个匹配的入口用主色渐变（突出推荐），其余用 outline 样式
- 底部保留一个「进入首页」兜底按钮
- 如果只领了 1 项，只显示 1 个主按钮 + 首页按钮
- 如果领了多项，最多显示 3 个入口按钮 + 首页按钮（避免按钮过多）

### 技术实现

#### 修改文件：`src/pages/Claim.tsx`

1. 新增一个映射常量，将 `grantedItems` 中的关键词映射到 `{ label, path, icon }` 对象
2. 在 success 状态的按钮区域，遍历 `grantedItems` 匹配关键词，生成对应入口按钮
3. 第一个按钮使用渐变主色样式，后续使用 outline
4. 保留「进入首页」作为最后一个 outline 按钮
